Getting Started with PROT-ON

The PROT-ON tool is designed to scan all possible interfacial mutations in order to identify critical amino acids for redesigning structurally characterized protein-protein interfaces (PPI). Traditionally, scanning mutations across protein interaction surfaces has been an experimentally costly process. To reduce the cost, computational methods have been developed to estimate the impact of an interfacial mutation on protein-protein interactions. PROT-ON uses EvoEF1 (active in both the web server and standalone versions) or FoldX (active only in the standalone version) to scan for mutations with the aim of finding the most mutable positions.

Clone the repository

For usage the stand-alone version of PROT-ON you should follow the below codes on command line. 

git clone https://github.com/CSB-KaracaLab/prot-on.git
cd prot-on

Environment setup

We strongly suggest creating a Python Virtual Environment (virtualenv) and activating it before the installation of PROT-ON dependencies. Also make sure receiving the system requirements, if not you can follow the link.

For this, please follow that codes on command line:

chmod +x set-env.sh
./set_env.sh

Then please enter the desired environment name at this stage, if you already have a python environment you can freely write that, script automatically prepare the python dependencies. As a result of running¬†setup.py, which is include the set_env.sh. PROT-ON will be ready to perform the mutational scan with EvoEF1 (January-2021 version of EvoEF1 comes in the PROT-ON package).

Please enter the desired environment name:

$environment_name

To run PROT-ON

You can execute the PROT-ON for first time with given PDB formatted complex that in example-input via:

example-input: complex.pdb

complex_chain_D_pssm.csv

python proton.py --pdb example-input/complex.pdb --chain_ID D

In this case; cut-off, IQR and algorithm definitions set the default formats that's are 5.0, 1.5 and EvoEF1 respectively.

The help page of PROT-ON can be called with:

python proton.py --help

Your Turn

The only requirement to run PROT-ON is a PDB file, containing a dimeric interface. You can supply your complex either by entering its PDB id or by directly uploading its coordinates. As PROT-ON allows the submission of complexes that have more than two chains, we ask the user to define the dimeric complex to be considered. We also ask the chain to be mutated, as PROT-ON can handle one chain at a time. In the current version, PROT-ON accepts only PDB-formatted files.

python proton.py --pdb <dir/filename of the structure> --chain_ID <chain ID of interest> --algorithm <EvoEF1/FoldX> --cut_off <cut-off to define the interface> --IQR <IQR rule to define outliers of box-and-whisker statistics>
